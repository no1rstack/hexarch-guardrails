## 🎯 THE 3 MAIN USE CASES

1. API BUDGET PROTECTION
   Prevent overspending on OpenAI, Anthropic, AWS, etc.
   
   @guardian.check("api_budget")
   def expensive_api_call():
       return openai.ChatCompletion.create(...)

2. RATE LIMITING
   Prevent getting rate-limited by Discord, OpenAI, etc.
   
   @guardian.check("rate_limit")
   async def send_discord_message(msg):
       await client.send(msg)

3. SAFE OPERATIONS
   Require confirmation for destructive operations.
   
   @guardian.check("safe_delete")
   def delete_file(path):
       os.remove(path)

## 🚨 ERROR HANDLING

from hexarch_guardrails import PolicyViolation

try:
    @guardian.check("policy_id")
    def my_func():
        pass
    
    my_func()
except PolicyViolation as e:
    print(f"Policy blocked: {e}")

## ❓ FREQUENTLY ASKED QUESTIONS

Q: Do I need OPA?
A: For the decorator-based SDK path, yes — use a reachable local or remote OPA server. For workflow and orchestration integrations, you can also use the built-in FastAPI service via `hexarch-ctl serve api`.

Q: How long does setup take?
A: ~5 minutes (3 lines of code + 1 YAML file + OPA running)

Q: Does it work with my favorite API?
A: Yes! It's policy-driven, not API-specific.

Q: Is this production-ready?
A: Yes! 30+ tests, full documentation, error handling.

Q: Can I change policies without redeploying?
A: Yes! Policies are in hexarch.yaml, change anytime.
